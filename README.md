## 4.1. Style Guidelines
4.1.1. General Style Guidelines.
	
- Branding
El logo principal está conformado por el icono de un tenedor y se encierra mediante circuitos indicando su vínculo con la tecnología, debajo del logo se encuentra el título “GASTROGO”.

IMAGEN -- IMGEN 

- Typography
La tipografía empleada será Roboto con sus variantes Regular y Bold. El tamaño de la letra varía entre 1 rem (16px), 1.5 rem (24px), 2 rem (32px) y 3 rem (48px).

Colors
El color principal que se utilizará es un tono turquesa (#1f99c2) y los colores secundarios son un tono verde (#00C9A7) y un tono blanco para contrastes más claros (#F5F9FF).

--IMAGEN--

Spacing
El spacing mantiene
Botones: padding de 1 rem (16px) vertical y 2 rem (32px) horizontal.
Margin entre texto 1 rem (16px).
Margin entre elementos 1.5 rem (24px).
Margin entre secciones 6 rem (72px).

Dimensión
El diseño debe adoptar un tono elegante, empleando colores profesionales como el turquesa y añadiendo un toque de entusiasmo con tonos verdes. Además, los íconos y formas se diseñarán de manera amigable para el usuario, utilizando en su mayoría bordes redondeados.

 
4.1.2. Web Style Guidelines.
	
Se empleó la tendencia de minimalismo funcional en el diseño web, utilizando el color blanco como base predominante en la mayoría de las secciones para ofrecer una visualización clara y un buen contraste entre los elementos, aunque también se podrá modificar a una versión nocturna para no molestar al lector.

## 4.2. Information Architecture

## 4.1.2. Organization Systems.

Para la organización visual del contenido, se aplicará el patrón de jerarquía visual para estructurar la información en las secciones. La relevancia de las frases se determinará por el tamaño de la fuente, siendo las de 48px (3 rem) las más destacadas, seguidas por las de 32px (2 rem), 24px (1.5 rem), y finalmente 16px (1 rem). 

En términos de esquemas de categorización, se utilizará una clasificación basada en la audiencia. También se aplicará una categorización cronológica en el registro de entradas, con las fechas más recientes apareciendo primero.

## 4.2.3. SEO Tags and Meta Tags

En el archivo index.html de nuestro proyecto Angular, hemos configurado varias etiquetas para mejorar la optimización en motores de búsqueda (SEO) y proporcionar metadatos relevantes:
Charset: <meta charset="UTF-8"> asegura la correcta visualización de caracteres especiales.
Compatibilidad: <meta http-equiv="X-UA-Compatible" content="IE=edge"> garantiza la mejor experiencia en navegadores como Internet Explorer.
Viewport: <meta name="viewport" content="width=device-width, initial-scale=1.0"> adapta el diseño a diferentes tamaños de pantalla.
Robots: <meta name="robots" content="index, follow"> indica a los motores de búsqueda que deben indexar la página y seguir los enlaces.
Favicon: <link rel="shortcut icon" href="assets/images/favicon.png" type="image/x-icon" /> define el icono de la página en la pestaña del navegador.
Title: <title>JobSync</title> establece el título de la página.
Keywords: <meta name="keywords" content="jobysnc, recursos humanos, software de proceso de contratación, software para RRHH, software para contratación"> incluye palabras clave para el SEO.
Description: <meta name="description" content="Atrae al mejor talento con el software altamente valorado de JobSync. Plataforma intuitiva que ahorra tiempo y agiliza el proceso de contratación. Comienza hoy."> proporciona una descripción que aparece en los resultados de búsqueda.

Cuando un cliente quiera buscar su producto según su estado, ya sea en algún restaurante específico o en una selección de estos.
Cuando un cliente realice una búsqueda del restaurante, lo hará basándose en la cercanía del restaurante.

## 4.2.5 Navigation Systems

### 4.7.2. Class Dictionary

-Restaurant:

Descripción: Representa una entidad de restaurante, incluyendo su nombre, dirección, disponibilidad y funciones para notificar y actualizar la disponibilidad.

-Customer:

Descripción: Representa a un cliente, incluyendo su ID, nombre y correo electrónico, con funcionalidades para registrarse, iniciar sesión, hacer pedidos y realizar reservas.

-Worker:

Descripción: Representa a un trabajador en el restaurante, incluyendo su ID, nombre y puesto, con funcionalidades para gestionar pedidos y reservas.

-Menu:

Descripción: Representa un menú, incluyendo su ID, nombre, descripción, precio y categoría. Incluye funcionalidades para agregar, eliminar y actualizar elementos del menú.

-MenuItem:

Descripción: Representa un elemento individual dentro de un menú, incluyendo su ID, nombre, descripción y precio.

-Order:

Descripción: Representa un pedido de un cliente, incluyendo el ID del pedido, la fecha del pedido, el estado y el monto total. Incluye funcionalidades para calcular el total y actualizar el estado del pedido.

-OrderItem:

-Descripción: Representa un ítem dentro de un pedido, incluyendo su ID, cantidad, precio y el ítem de menú asociado. Incluye una función para calcular el subtotal.

-Reservation:

Descripción: Representa una reserva de mesa, incluyendo su ID, fecha de reserva, número de personas y ID de la mesa. Incluye funcionalidades para crear, cancelar y actualizar reservas.

-OrderList:

Descripción: Representa una colección de pedidos, con funcionalidades para agregar, eliminar y obtener pedidos.

-ReservationList:

Descripción: Representa una colección de reservas, con funcionalidades para agregar, eliminar y obtener reservas.

-PaymentMethod:
